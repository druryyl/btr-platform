# M32.3 — Monthly History (L1) + Trend — Implementation Summary

**Milestone:** M32.3  
**Date:** 2026-06-24  
**Status:** Complete  
**Prerequisite:** M32.1 platform foundation, M32.2 Customer Performance Profile (L0), M32.2R hardening

---

## Objective

Deliver the first historical layer of Entity Analytics: generic L1 monthly snapshots, month-close retention, Trend Engine read path, and Customer Profile Trend visualization — reusable by all future entity types.

---

## Pipeline Delivered

```text
Customer Dashboard Aggregation (M17–M31)
  → CustomerEntityAnalyticsProducer
      → L0 CURRENT (ReplaceCurrentMetrics)
      → EntityAnalyticsMonthCloseService (freeze prior month + retention purge)
      → L1 MONTHLY (SaveMonthlyHistory — TrendEligible KPIs only)
  → EntityTrendEngine (profile read)
  → Profile Trend section + KpiTrendChart (portal)
```

## 1. SQL Changes

| Artifact | Path |
| -------- | ---- |
| L1 monthly table | `btr.sql/Tables/ReportingContext/BTRPD_EntityAnalytics_Monthly.sql` |
| Month-close meta | `btr.sql/Tables/ReportingContext/BTRPD_EntityAnalytics_MonthClose.sql` |
| Upgrade script | `btr.sql/Scripts/Upgrade_EntityAnalytics_M32.sql` (appended `:r` includes) |
| Greenfield install | `btr.sql/Scripts/Create_BTRPD_PortalDashboard_Tables.sql` |

**L1 grain:** `(EntityType, EntityId, PeriodYear, PeriodMonth, KpiId)` — unique index enforces one row per entity per month per KPI.

**Period semantics:** `PeriodSemantics` column stored at write time (`MTD`, `AllTimeOpen`, etc.) from KPI registry.

**Month-close:** `IsClosed` on L1 rows; `BTRPD_EntityAnalytics_MonthClose` tracks frozen periods per entity type.

---

## 2. Repository Changes

Extended `IEntityAnalyticsMonthlyRepository` / `EntityAnalyticsRepository`:

| Method | Behavior |
| ------ | -------- |
| `GetHistory` | Range query with optional from/to period filters |
| `GetLatestPeriods` | Distinct periods for entity, ordered chronologically |
| `SaveMonthlyHistory` | MERGE upsert; skips rows where `IsClosed = 1` |
| `CloseMonth` | Sets `IsClosed = 1` on all L1 rows + inserts MonthClose meta |
| `IsMonthClosed` | Checks MonthClose table |
| `PurgeHistoryOlderThan` | Deletes rows older than `HistoryRetentionMonths` |

`GetMonthlyMetrics` / `GetMonthlyRange` delegate to `GetHistory`.

---

## 3. Producer Changes

`CustomerEntityAnalyticsProducer` now:

1. Writes L0 (unchanged semantics)
2. Calls `IEntityAnalyticsMonthCloseService.EnsurePriorMonthClosed`
3. Upserts L1 for current `(BusinessDate.Year, BusinessDate.Month)`

**Shared KPI mapping** via `BuildTrendKpiValues()` — same portfolio row fields for L0 and L1 (no duplicate business calculations).

**Trend KPIs written:** CU-KPI-009, CU-KPI-010, FI-KPI-013 (all `TrendEligible = true`).

---

## 4. Worker Integration

`RefreshDashboardCustomerSnapshotWorker` — single orchestrator call inside existing `TransHelper` scope:

- Progress step label: `"Produce entity analytics L0+L1 snapshot"`
- No new scheduler or separate worker

---

## 5. Trend Engine Design

`EntityTrendEngine` (`IEntityTrendEngine`):

- Resolves entity KPI pack; filters `TrendEligible` KPIs
- Loads L1 via `GetHistory` over last `min(12, HistoryRetentionMonths)` periods
- Groups by KPI ID; orders points chronologically
- Builds `ProfileTrendSectionDto` with `PeriodSemantics`, `Unit`, `IsClosed`, `PeriodLabel`
- Does **not** compute business KPIs or MoM/YoY (deferred to Comparison engine M32.8)

`EntityPerformanceProfileComposer` injects trend engine; `Trend.IsAvailable = true` when series exist.
