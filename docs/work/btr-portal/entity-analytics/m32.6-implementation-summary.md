# M32.6 — Relationship Engine (L4) — Implementation Summary

**Milestone:** M32.6  
**Date:** 2026-06-24  
**Status:** Complete  
**Prerequisite:** M32.1 platform foundation through M32.5 Attention History (L3)

---

## Objective

Deliver generic L4 relationship snapshots and the Related Entities section on Entity Performance Profiles — answering *"What entities explain this customer's behavior?"* without querying transactional tables at profile read time.

---

## Pipeline Delivered

```text
Customer Dashboard Aggregation (M17–M31)
  → CustomerMtdItemRollupDal + DashboardCustomerRelationshipAggregator (refresh-time)
  → CustomerEntityAnalyticsProducer
      → L0 CURRENT
      → L1 MONTHLY
      → EntityRankingEngine (L2)
      → EntityAttentionEngine (L3)
      → EntityRelationshipEngine (L4)
  → EntityRelationshipEngine (profile read)
  → Profile Related Entities section (portal)
```

### Lifecycle rules

| Rule | Decision |
|------|----------|
| Write path | Producer emits snapshots; engine applies Top-N and persists |
| Replace semantics | Delete all rows for `(SourceEntityType, PeriodYear, PeriodMonth)` then insert |
| Rank assignment | Descending `MetricValue`; 1-based rank within relationship code |
| Closed month | Skip L4 write when `IsMonthClosed` |
| Profile read | L4 only — no DAL calls at HTTP time |

**Customer relationship pack (`customer-relationships`):**

| Code | Label | Target | TopN |
|------|-------|--------|------|
| `AssignedSalesman` | Assigned Salesman | Salesman | 1 |
| `TopItemsByOmzet` | Top Items | Item | 10 |
| `TopPrincipalsByOmzet` | Top Principals | Supplier | 10 |

## 4. Producer Integration

`CustomerEntityAnalyticsProducer` builds `EntityRelationshipSnapshot` rows and calls `EntityRelationshipEngine.PersistRollups` after L3.

Producer does **not** write L4 tables directly.

Worker progress label: `"Produce entity analytics L0+L1+L2+L3+L4 snapshot"`.
