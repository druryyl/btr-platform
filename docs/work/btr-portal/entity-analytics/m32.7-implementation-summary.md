# M32.7 — Comparison Engine — Implementation Summary

**Milestone:** M32.7  
**Date:** 2026-06-24  
**Status:** Complete  
**Prerequisite:** M32.1–M32.6 (L0–L4 snapshots and read engines)

---

## Objective

Deliver a read-time Comparison Engine for cross-period (profile) and multi-entity (compare page) analysis, plus entity search for the comparison picker — without new snapshot tables or transactional reads.

---

## Pipeline Delivered

```text
GET /api/entity-analytics/compare
GET /api/entity-analytics/search
        ↓
EntityComparisonEngine (read-time composition)
        ↓
L0 CURRENT · L1 MONTHLY · L2 RANKING · L3 ATTENTION · L4 RELATIONSHIP
        ↓
EntityTrendEngine / EntityRankingEngine / EntityAttentionEngine / EntityRelationshipEngine (delegated)
        ↓
CustomerCompareView.vue + ProfileComparisonSection (MoM/YoY)
```

| Component | Path |
| --------- | ---- |

| `ComparisonMode` | `Models/ComparisonMode.cs` |

### Modes (M32.7)

| Mode | Consumer | Data sources |
| ---- | -------- | ------------ |
| `CrossPeriod` | Profile `Comparison` section | L0 current vs L1 prior month + prior year |
| `MultiEntity` | Compare API + `CustomerCompareView` | Batch L0 + delegated L1–L4 engines per entity |

**Deferred to M32.8:** `Peer`, `Average`, `Radar` modes (require L5).

### MoM / YoY derivation

- Calculated at read time in `EntityComparisonCalculator` — not persisted
- Growth % = `(current - prior) / abs(prior) * 100`; null when prior is 0 or missing
- Partial-month labels follow `EntityTrendEngine` MTD convention

## 2. API Examples

### Compare (multi-entity)

```http
GET /api/entity-analytics/compare?entityType=Customer&entityIds=C001,C002,C003&kpiIds=CU-KPI-009,CU-KPI-010
Authorization: Bearer <token>
```

**Response (abbreviated):**

```json
{
  "EntityType": "Customer",
  "ContractVersion": "M32.7-v1",
  "Entities": [
    { "EntityCode": "C001", "DisplayName": "Alpha Corp", "GeneratedAt": "2026-06-24T08:00:00" }
  ],
  "KpiComparison": { "IsAvailable": true, "Rows": [] },
  "TrendComparison": { "IsAvailable": true, "Overlays": [] },
  "RankingComparison": { "IsAvailable": true, "Entities": [] },
  "AttentionComparison": { "IsAvailable": true, "Entities": [] },
  "RelationshipComparison": { "IsAvailable": true, "Entities": [] },
  "Warnings": []
}
```

**Guards:** 2–5 entities, same `EntityType`, enabled type only.

### Search (entity picker)

```http
GET /api/entity-analytics/search?entityType=Customer&q=alpha&top=10
Authorization: Bearer <token>
```

```json
{
  "EntityType": "Customer",
  "Results": [
    {
      "EntityCode": "C001",
      "DisplayName": "Alpha Corp",
      "ProfileRoute": "/analytics/customers/C001"
    }
  ]
}
```

### Profile Comparison section (cross-period)

`GET /api/entity-analytics/Customer/C001` now returns `Comparison.Metrics[]` with `MomGrowthPercent`, `YoyGrowthPercent`, and period labels. `ContractVersion: M32.7-v1`.

---

## 3. Search API Design

- **Source:** L0 `BTRPD_EntityAnalytics_Current` only (snapshot-backed)
- **Match:** `EntityCode`, `EntityId`, or `EA-META-DISPLAY-NAME` text
- **Entity-type agnostic:** no master DAL dependency
- **Min query length:** 2 characters
- **Default top:** 10 (max 25)
