**Milestone:** M32.1

## Objective

Deliver reusable Entity Analytics platform infrastructure: generic domain model, L0 CURRENT snapshot schema, repository/DAL, KPI registry shell, profile composer, producer/worker extension points, API, DI, and Vue profile shell — without entity-specific KPI logic or L1–L5 layers.

## Assumptions

1. **Refined milestone sequence** — L1–L5 and analytic engines deferred to M32.3+ per clarified scope; original implementation plan M32.1 engine list not implemented here.
2. **L0 only** — Single generic CURRENT table; no monthly/ranking/attention/relationship/radar tables yet.
3. **Empty KPI registry** — No customer-default pack or catalog KPI instances; registry is structural only.
4. **No worker wiring** — `EntityAnalyticsProducerOrchestrator` registered but domain snapshot workers unchanged until M32.2.
5. **Synthetic identity** — When no L0 rows exist, profile overview uses route `entityId` as display identity.
6. **No sidebar menu** — Routes registered; navigation menu entry deferred to M32.2.
7. **ReportingContext convention** — Platform models live in `btr.application`, not `btr.domain`.
