# M32.4 — Ranking History (L2) — Implementation Summary

**Milestone:** M32.4  
**Date:** 2026-06-24  
**Status:** Complete  
**Prerequisite:** M32.1 platform foundation, M32.2 Customer Profile (L0), M32.3 Monthly History + Trend (L1)

---

## Objective

Deliver generic L2 ranking snapshots and rank history on Entity Performance Profiles — answering *"What was this entity's rank over time?"* rather than Top-N dashboard lists.

---

## Pipeline Delivered

```text
Customer Dashboard Aggregation (M17–M31)
  → CustomerEntityAnalyticsProducer
      → L0 CURRENT (ReplaceCurrentMetrics)
      → EntityAnalyticsMonthCloseService (freeze prior month + retention purge)
      → L1 MONTHLY (SaveMonthlyHistory — TrendEligible KPIs)
      → EntityRankingEngine.ComputeAndPersistRanks (L2 — RankEligible KPIs)
  → EntityRankingEngine (profile read)
  → Profile Ranking section + KpiRankChart (portal)
```

## 2. Ranking Engine Design

| Component | Path |
| --------- | ---- |
| `IEntityRankingEngine` | `Contracts/IEntityRankingEngine.cs` |
| `EntityRankingEngine` | `Services/EntityRankingEngine.cs` |
| `EntityRankingCalculator` | `Services/EntityRankingCalculator.cs` (pure ranking logic) |

### Ranking rules

| Rule | Decision |
|------|----------|
| Sort basis | L1 `NumericValue` for the period |
| Direction | From KPI metadata `Direction` (exposed as `RankingDirection` on profile DTOs) |
| Population | Active entities only (`EA-META-IS-ACTIVE = 1`) |
| Null values | Excluded — no L2 row |
| Tie handling | **Competition ranking** (1, 2, 2, 4) |
| Percentile | `100 × (PopulationSize - RankPosition + 1) / PopulationSize` |
| Closed month | Skip L2 upsert when `IsMonthClosed` |
| KPI selection | `RankEligible = true` from registry — no hardcoded IDs |

**Customer rank-eligible KPIs:** CU-KPI-009 (HigherIsBetter), CU-KPI-010 (LowerIsBetter).

## 3. Repository Changes

Extended `IEntityAnalyticsRankingRepository` / `EntityAnalyticsRepository`:

| Method | Behavior |
| ------ | -------- |
| `SaveRankingHistory` | MERGE upsert on unique key |
| `GetRankingHistory` | Range query with optional from/to period |
| `GetLatestRanking` | Most recent period for entity + KPI |
| `GetRankingPeriods` | Distinct periods for history window |
| `GetPeriodMetricsForPopulation` | L1 values joined with L0 IsActive for ranking pass |

`PurgeHistoryOlderThan` now also purges L2 rows beyond retention window.

---

## 4. Producer Integration

`CustomerEntityAnalyticsProducer` calls `_rankingEngine.ComputeAndPersistRanks` after `SaveMonthlyHistory`.

Worker progress label: `"Produce entity analytics L0+L1+L2 snapshot"`.

No duplicate KPI calculation — engine reads persisted L1 values.
