# M32.5 — Attention History (L3) — Implementation Summary

**Milestone:** M32.5  
**Date:** 2026-06-24  
**Status:** Complete  
**Prerequisite:** M32.1 platform foundation, M32.2 Customer Profile (L0), M32.3 Monthly History (L1), M32.4 Ranking History (L2)

---

## Objective

Deliver generic L3 attention lifecycle snapshots and Attention History on Entity Performance Profiles — answering *"What problems has this entity experienced, when did they start, and are they still active?"* without querying transactional tables at profile read time.

---

## Pipeline Delivered

```text
Customer Dashboard Aggregation (M17–M31)
  → CustomerEntityAnalyticsProducer
      → L0 CURRENT (ReplaceCurrentMetrics)
      → EntityAnalyticsMonthCloseService (freeze prior month + retention purge)
      → L1 MONTHLY (SaveMonthlyHistory)
      → EntityRankingEngine.ComputeAndPersistRanks (L2)
      → EntityAttentionEngine.DiffAndPersistSignals (L3)
  → EntityAttentionEngine (profile read)
  → Profile Attention History section (portal)
```

### Lifecycle rules

| Rule | Decision |
|------|----------|
| New signal | Create row; `FirstSeen` = `LastSeen` = current period; streak = 1 |
| Same-month re-refresh | Touch metadata only — no occurrence increment |
| Consecutive month | `ConsecutivePeriods++`, `TotalOccurrences++` |
| Gap month (while active) | Reset `ConsecutivePeriods` to 1; `TotalOccurrences++` |
| Signal absent | `IsActive = false`; `LastSeen` frozen |
| Reappearance after resolved | `IsActive = true`; new streak = 1; `TotalOccurrences++`; original `FirstSeen` preserved |
| Closed month | Skip L3 write when `IsMonthClosed` |
| Entity isolation | All operations keyed by `(EntityType, EntityId, SignalCode)` |

**Customer signal catalog (M17 keys):** `Overdue`, `Dormant`, `PlafondBreach`, `SuspendedWithSales`.

## 4. Producer Integration

`CustomerEntityAnalyticsProducer` builds normalized `EntityAttentionSignalSnapshot` rows per portfolio customer (including empty signal lists for resolution diff) and calls `EntityAttentionEngine.DiffAndPersistSignals` after L2.

Producer does **not** write L3 tables directly.

Worker progress label: `"Produce entity analytics L0+L1+L2+L3 snapshot"`.

## Follow-up Recommendations for M32.6

1. **Relationship Engine** should follow the same producer-orchestrated pattern: producer emits normalized relationship inputs; `EntityRelationshipEngine` owns L4 lifecycle writes.
2. **L3 retention policy** — consider whether resolved signals older than N months should be purged (currently retained indefinitely; L1/L2 purge does not touch L3).
3. **Salesman/Supplier/Item packs** — register attention signal catalogs per entity type (M32.9–M32.11) without changing `EntityAttentionEngine`.
4. **Profile cross-link** — optional future enhancement: link active attention signals to Alert Center / CU01 evidence routes (out of M32.5 scope).

Customer Dashboard → Customer Producer → L0 → L1 → L2 → L3 Attention Engine → Customer Profile Attention History forms a reusable attention-lifecycle pipeline. Future entity types adopt the same L3 infrastructure via producer registration and signal catalog mapping only.
