## 1. Implementation Summary

Delivered **Item** as the fourth Entity Analytics consumer by mirroring the M32.10 Supplier entity-pack pattern: registrar, attention/relationship catalogs, producer (L0–L5 orchestration), worker hook on `InventoryRisk`, IN02 `ProfileRoute` links, frontend profile/compare routes, and reconciliation tests — with **no platform engine or L0–L5 schema changes**.

```text
RefreshDashboardInventoryRiskSnapshotWorker
  → DashboardInventoryRiskAggregator (M19)
  → DashboardInventoryForecastAggregator (M28)
  → DashboardInventoryOptimizationAggregator (M28.5)
  → DashboardItemPortfolioBuilder + ISalesmanMtdItemRollupDal
  → DashboardItemRelationshipAggregator
  → BTRPD_InventoryRisk snapshot save
  → EntityAnalyticsProducerOrchestrator
        → ItemEntityAnalyticsProducer
              L0 CURRENT  (EntityId = BrgId, EntityCode = BrgCode)
              L1 MONTHLY  (TrendEligible KPIs; ADR-EA-011 gate)
              L2 RANKING  (RankEligible KPIs; peer = item-category)
              L3 ATTENTION (M19 + M28 signal keys via EntityAttentionEngine)
              L4 RELATIONSHIP (Top customers/salesmen + primary supplier)
              L5 RADAR     (peer group item-category)
  → GET profile / search / compare (existing platform API + Vue shells)
```

| KPI ID | Category | Profile role | Trend | Rank | Radar |

| **IN-KPI-020** | Risk | Days of Supply | ✓ | ✓ | Axis 3 — Velocity |
| **IN-KPI-021** | Activity | Recommended Purchase Qty | ✓ | — | — |
| **EA-RADAR-GROWTH-MOM** | Growth | MoM inventory value growth | — | — | Axis 2 (`RadarSourceKpiId = IN-KPI-001`) |
| **EA-DIM-CUSTOMER-COUNT** | Portfolio | Distinct MTD buyers | — | — | Axis 4 — Market Reach |
| **EA-RADAR-ATTENTION-RISK** | Risk | Active attention count | — | — | Axis 5 |
| **EA-DIM-QTY-ON-HAND** | Volume | On-hand qty | — | ✓ | Axis 6 — Supplier Strength |

**Overview dimensions (L0 meta):** Category, Supplier, Movement Class, Days Since Last Faktur, Active, Attention Signals.

**Not in profile KPI pack:** IN02 company headline cards (Dead Stock Count, At-Risk %). Per-SKU risk surfaces via **L3 attention signals**.

**Evidence routes:** `/reports/inventory` with filter dimension `brgCode`.

**Excluded (no existing dashboard rules):** Margin Decline, High Return.

## 3. Producer Design (L0–L5)

| Layer | Source | Entity keys |
| ----- | ------ | ----------- |
| **L0** | `DashboardItemPortfolioRow` list (item groups + forecast + rollups) | `EntityId = BrgId`, `EntityCode = BrgCode` |
| **L1** | TrendEligible KPI values for current `(Year, Month)` when `IsTrendEligible` | ADR-EA-011: stock > 0 OR sale in 24 months |
| **L2** | RankEligible KPIs across active items | Peer group `item-category` |
| **L3** | M19 `AttentionList` + M28 `TopRisks` grouped by `BrgId` | M19/M28 signal constants |
| **L4** | `DashboardItemRelationshipAggregator` output | Pack `item-relationships` |
| **L5** | `PeerGroupRuleId = item-category` | Six radar axes from registrar metadata |

**KPI mapping (L0 numeric rows)**

| KPI ID | Portfolio field |
| ------ | ----------------- |
| IN-KPI-001 | `InventoryValue` |
| IN-KPI-020 | `DaysOfSupply` |
| IN-KPI-021 | `RecommendedPurchaseQty` |
| EA-DIM-QTY-ON-HAND | `Qty` |

## 7. API Examples

```json

  "EnabledEntityTypes": [ "Customer", "Salesman", "Supplier", "Item" ]
```

```http
GET /api/entity-analytics/Item/{brgCode}
```

**Search**

```http
GET /api/entity-analytics/search?entityType=Item&q=widget
```

**Compare**

```http
GET /api/entity-analytics/compare?entityType=Item&entityIds=BRG1,BRG2
```
