**Milestone:** M32.2

## Objective

Deliver the first concrete consumer of the Entity Analytics Platform: **Customer Performance Profile** at L0 CURRENT only, reusing M17/M31 aggregator outputs without duplicate KPI calculations.

---

## What Was Implemented

### Customer KPI Pack (`customer-default`)

| KPI ID | Category | Source field | Reconcile with |
|--------|----------|--------------|----------------|
| CU-KPI-009 | Financial | `MtdOmzet` | CU01 Top Omzet row; CU05 Customer Report |
| CU-KPI-010 | Financial | `OpenBalance` | CU01 Top Piutang row; CU05 Customer Report |
| FI-KPI-013 | Risk | `OverdueBalance` | CU05 `OverdueBalance`; CU01 attention `Overdue` |

Overview dimensions (not catalog KPI rows): Wilayah, Klasifikasi, Salesman, Lifecycle, Tier, Portfolio Action, M29 Category, Attention Signals, Last Purchase, Faktur Count (6 Mo), Active MTD, Portfolio Priority Score.

### Customer Producer

`CustomerEntityAnalyticsProducer` maps in-memory `DashboardCustomerPortfolioAggregateResult.Customers` (full master universe) plus M17 attention signals into `BTRPD_EntityAnalytics_Current`.

- **No DAL calls** — reads `CustomerEntityAnalyticsProduceInput` attached to `EntityAnalyticsProduceContext.DomainInput`
- **Entity key:** `CustomerCode` (CustomerCode-first)
- **Meta rows:** `EA-META-*` and `EA-DIM-*` for identity/overview

### Worker Integration

`RefreshDashboardCustomerSnapshotWorker` calls `EntityAnalyticsProducerOrchestrator.ProduceForDomain("Customer", ...)` immediately after domain snapshot save (`Customer:EntityAnalytics` progress step).

### Profile API

`GET /api/entity-analytics/Customer/{customerCode}` returns populated:

- **Overview** — identity + dimensions from L0 meta rows
- **KPI Summary** — catalog KPI envelopes from L0
- **Evidence** — Sales / Piutang / Customer reports with `?customerCode=` filter

Other sections: `IsAvailable = false`, `UnavailableReason = "NotImplemented"`.

### Dashboard Integration

- `ProfileRoute` on CU01 attention and ranking rows → `/analytics/customers/{code}`
- `ProfileRoute` on CU04 priority queue rows
- Customer Analytics: ranking row click → profile; attention list Profile button + code link
- Customer Portfolio: Profile link in priority table

### Portal UI

- Route: `/analytics/customers/:customerCode` → redirects to generic profile route
- Production sections: Overview, KPI Summary, Evidence
- Placeholder sections preserved (Comparison, Trend, Radar, Ranking, Attention, Related Entities)
