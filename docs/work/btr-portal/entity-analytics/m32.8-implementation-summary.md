**Milestone:** M32.8  
**Date:** 2026-06-24  
**Status:** Complete  
**Prerequisite:** M32.1–M32.7 (L0–L4 snapshots, Comparison Engine)

## 1. Implementation Summary

Delivered the generic L5 Radar Engine with peer-group percentile normalization at refresh, Customer radar pack (six orthogonal axes), profile Radar section, and compare-page radar overlay — all reading precomputed L5 snapshots only.

```text
CustomerEntityAnalyticsProducer
  → L0–L4 (unchanged)
  → EntityRadarEngine.ComputeAndPersistScores → L5
        ↓
GET profile / GET compare
  → EntityRadarEngine.BuildRadarSection / EntityComparisonEngine.BuildRadarComparison
  → reads BTRPD_EntityAnalytics_Radar only
```

## 3. Radar Engine Design

| Component | Path |
| --------- | ---- |
| `IEntityRadarEngine` | `Contracts/IEntityRadarEngine.cs` |
| `EntityRadarEngine` | `Services/EntityRadarEngine.cs` |
| `EntityRadarCalculator` | `Services/EntityRadarCalculator.cs` |
| `PeerGroupResolver` | `Services/PeerGroupResolver.cs` |

### Normalization (ADR-EA-007)

| Rule | Behavior |
| ---- | -------- |
| Primary | Peer-percentile (0–100) within declared peer group |
| Direction | `HigherIsBetter` descending; `LowerIsBetter` ascending |
| Missing data | Axis omitted — never scored as 0 |
| Band fallback | When peer group &lt; 10 and `NormalizationRule = AchievementBand` |
| Minimum peer group | No L5 write / section hidden when &lt; 5 |

### Value resolution (`RadarValueSource`)

| Source | Layer |
| ------ | ----- |
| `L0Kpi` | L0 numeric KPI |
| `L1MomGrowthPercent` | L1 MoM % via `EntityComparisonCalculator` |
| `L0DimensionNumeric` | L0 dimension text parsed to decimal |
| `L3ActiveSignalCount` | L3 active signal count |

## 4. Peer Group Design

| `PeerGroupRuleId` | Membership |
| ----------------- | ---------- |
| `customer-wilayah` | Active customers with same `EA-DIM-WILAYAH` |
| `salesman-all-active` | All active reps |
| `item-category` | Same category dimension |
| `supplier-all-active` | All active principals |

`PeerGroupResolver` builds a peer index at refresh; `EntityRadarEngine` skips entities in groups smaller than 5.

## 5. Customer Radar Pack (v1)

Six axes — **no Ranking axis** (avoids double-counting with Revenue):

| Order | Axis | KPI ID | Source |
| ----- | ---- | ------ | ------ |
| 1 | Revenue | `CU-KPI-009` | L0 omzet |
| 2 | Growth | `EA-RADAR-GROWTH-MOM` | L1 MoM on CU-KPI-009 |
| 3 | Stability | `CU-KPI-010` | L0 open balance (LowerIsBetter) |
| 4 | Portfolio Strength | `EA-DIM-PORTFOLIO-PRIORITY-SCORE` | L0 dimension |
| 5 | Attention Risk | `EA-RADAR-ATTENTION-RISK` | L3 active signal count |
| 6 | Engagement | `EA-DIM-FAKTUR-COUNT-6MO` | L0 dimension |

Registered in `CustomerEntityAnalyticsRegistrar` via `RadarEligible`, `RadarAxisOrder`, `RadarValueSource`, `RadarDisplayName`.

## 6. API Changes

**Contract version:** `M32.8-v1`

### Profile

`GET /api/entity-analytics/Customer/{code}` — `Radar` section now populated from L5:

```json
"Radar": {
  "IsAvailable": true,
  "PeerGroupRuleId": "customer-wilayah",
  "PeerGroupSize": 12,
  "PeriodLabel": "Jun 2026 (MTD)",
  "Axes": [
    { "KpiId": "CU-KPI-009", "DisplayName": "Revenue", "Score": 86.5, "Direction": "HigherIsBetter" }
  ]
}
```

When peer group &lt; 5: `UnavailableReason: PeerGroupTooSmall` with `UnavailableExplanation`.

### Compare

`GET /api/entity-analytics/compare` — new sections:

- `RadarComparison` — multi-entity overlay from L5 batch read
- `PeerComparison` — primary entity peer radar context from L5

`ComparisonMode.Peer` and `ComparisonMode.Radar` added to enum.
